# 一、智慧生产模块到底做什么

这个模块只做两件事：

## 1）编排调整

对已有编排做最简单参数修改，比如：

- 每天 3 次改成 2 次
- 每次 20 分钟改成 15 分钟
- 08:00 改成 09:00

**不做复杂规则引擎，不从零创建复杂编排。**

------

## 2）数据查看

只查看当前资源的生产数据：

- 当前温度、湿度、光照、土壤湿度
- 当前摄像头状态
- 视频流地址 / 最近画面

原始文档就是这么定义的。

# 七、智慧生产模块最终业务规则

------

## 访问规则

1. 只有租赁有效的资源才能进入智慧生产模块
2. 只有当前订阅人本人可以查看和修改
3. 如果订阅状态不是 `ACTIVE`，整个模块不可用
4. 生产模块不自己处理订阅、续订、退订业务，这些都属于租赁模块

------

## 编排规则

1. 只允许修改简单参数：
   - 每日次数
   - 每次时长
   - 执行时间点
2. 不允许新增复杂规则
3. 不允许做设施、设备变更
4. 只允许租赁有效时修改编排
5. 只能修改白名单字段

------

## 查看规则

1. 只有租赁有效时才能查看环境信息
2. 只有租赁有效时才能查看摄像头
3. 摄像头离线时返回离线状态，不抛业务异常
4. 环境数据只取最新一条快照，不做复杂历史分析

------

# 十、状态机

------

## 编排状态机

原始方案建议：

```text
ENABLED -> DISABLED
DISABLED -> ENABLED
```

因为用户可能需要暂停。

最终版继续保留这个状态机。

------

## 摄像头状态机

```text
ONLINE -> OFFLINE
ONLINE -> ERROR
OFFLINE -> ONLINE
ERROR -> ONLINE
```

这只是展示状态，不参与核心业务。

------

## 模块可用状态

最终版不单独建表，所以不再维护 `production_access` 的状态机。
模块是否可用，直接取决于租赁模块里的订阅状态：

```text
subscription.ACTIVE   -> 可用
subscription.CANCELLED / EXPIRED -> 不可用
```
