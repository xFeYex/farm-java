# 一、总体系统定位（统一版）

本系统拆成 **三个独立但可联动的子域（Domain）**：

1. **农场租赁域（Resource Domain）**
   👉 管“资源从发布 → 被租 → 生命周期”
2. **智慧生产域（Production Domain）**
   👉 管“租了之后能干什么（编排 + 数据查看）”
3. **农产品交易域（Product Domain）**
   👉 管“生产结果如何卖出去”

------

## 核心设计原则（你这套系统最关键）

来自你三份文档统一抽象：

- 极简 MVP（全部模块都只保主链路）
- 不做用户体系、不做支付、不做后台
- 状态机必须固定（避免后期爆炸）
- 模块之间**只通过“状态同步”联动，不直接耦合逻辑**

# 三、模块职责划分（非常关键）

## 1️⃣ 农场租赁域（基础域）

👉 系统“入口域”

**负责：**

- 发布资源
- 订阅（租赁）
- 续订 / 退订
- 资源状态

**输出给其他域：**

- Subscription 状态（ACTIVE / EXPIRED / CANCELLED）

------

## 2️⃣ 智慧生产域（依赖租赁）

👉 只能在“租了之后”使用

**依赖：**

- `subscription.status = ACTIVE`

**负责：**

- 编排参数调整
- 环境数据查看
- 摄像头查看
- 访问控制（production_access）

------

## 3️⃣ 农产品交易域（独立变现域）

👉 和租赁“弱关联”，但业务上相关

**负责：**

- 商品上架
- 购买
- 发货
- 收货
- 退货
- 结算（7天规则）

------

# 四、统一业务主链路（跨模块）

这是你系统最重要的一条线👇

------

## 🌱 主流程（完整闭环）

```text
发布资源
   ↓
用户订阅（租赁）
   ↓
生产模块开放（ACTIVE）
   ↓
用户查看数据 / 调整编排
   ↓
产出农产品
   ↓
卖方上架商品
   ↓
买方购买
   ↓
发货 → 收货
   ↓
7天后结算
```

# 六、跨模块关键联动

## 1️⃣ 租赁 → 生产（唯一强依赖）

```text
Subscription.ACTIVE → ProductionAccess.ACTIVE
Subscription.EXPIRED → 禁用生产模块
```

👉 生产域不判断租赁逻辑，只接收结果

------

## 2️⃣ 生产 → 交易（弱关系）

```text
生产 → 农产品产出 → 手动上架
```

------

## 3️⃣ 交易内部（最复杂）

👉 结算规则：

```text
收货 + 7天 + 无退货 = SETTLED
```
